Concurrent Programming C
Philosophers
This projected aimed at creating a solution to the classic concurrency puzzle: The Dining Philosophers Problem. Imagine five philosophers sitting around a table, each needing two forks (shared with neighbours) to eat but only one at a time is allowed to pick them up. The challenge was in designing a system where they can eat without deadlock (everyone waiting forever) or starvation (someone never eating), which made use of threads and mutexes.

Signal Processing C
Minitalk
This projects amounts to a communication program. The program enables a message to be sent from a client binary to a server binary through the use of only signals. That implies that a text message is first encoded, on the "client side", to bits - for it to be sent using SIGUSR1/SIGUSR2 - and is then decoded on the "server side" from bits back to ASCII for it to be printed on the terminal.
